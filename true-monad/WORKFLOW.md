# True Monad 本地工作流

本文件定义已经安装到当前项目的固定运行规则。外部 Skill 只能负责首次安装、检测和显式升级；日常工作不得使用外部包中的脚本替代本目录代码。

## 工作区

- `next/`：用户原始指令与批复。原始内容不可改写。
- `current/`：本轮理解、认知环境和结构化状态。
- `end/`：本轮给用户的持久化 Markdown 结果。
- `archive/`：经过校验的历史轮次，进入 Git。
- `service/`：项目固定版本的工作台 Service。
- `scripts/`：项目固定版本的校验、归档与迁移脚本。

## 执行顺序

1. 确认本轮用户原始指令已经进入 `next/`。
2. 更新本轮指令理解和 Compressed Monad。
3. 执行任务并写入 end 与 current 状态。
4. 校验 current CSV 和 next 文件命名。
5. 用户或工作台明确触发归档后，运行项目本地归档脚本。
6. 归档成功后保留 Compressed Monad，清理其他未归档状态。

## 安全规则

- 检测到 `manifest.json` 时，安装器必须拒绝默认覆盖。
- 存在未归档指令、状态或产出时，不得升级或强制重装。
- Git 回滚不能恢复被 `.gitignore` 忽略的未归档数据；升级前必须先归档或另行完整备份。
